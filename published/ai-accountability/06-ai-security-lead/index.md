# AI Security Lead

A model can sit behind perfectly locked-down infrastructure and still be talked into leaking data or taking an action it shouldn't — through the prompt itself. The **AI Security Lead** exists to find that failure mode before an attacker does: adversarial testing and model-level security, distinct from the infrastructure security a CISO already owns. This bundle is the operator's manual for that seat.

It's the sixth and final role in [AI Accountability](/ai-accountability/index.md)'s initial map — distinct from [AI Risk Officer](/ai-accountability/05-ai-risk-officer/index.md) (non-adversarial accuracy and drift) and [AI Governance Lead](/ai-accountability/04-ai-governance-lead/index.md) (policy and sign-off): this seat supplies the technical finding that those functions act on.
