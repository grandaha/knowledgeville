The Core Framework tells you what matters and why. These guides tell you how to actually do the work. Each one covers a specific discipline in full operational detail: how to run the process, which tools to use and why, sequencing and common failure points, and a practical checklist you can hand to whoever owns the work.

They are written for people who already understand the problem and need to execute — not for diagnosis. If you are still determining where your biggest gaps are, start with the [Assessment & Measurement](/enterprise-ai-transformation/tracks/03-data-readiness/04-assessment-and-measurement/index.md) section first.

| Guide | When to reach for it |
| --- | --- |
| [Data Audits & Automated Quality Governance](/enterprise-ai-transformation/tracks/03-data-readiness/03-practitioner-guides/data-audits-and-automated-quality-governance.md) | You need to baseline current data quality and set up ongoing automated monitoring |
| [Master Data Management (MDM)](/enterprise-ai-transformation/tracks/03-data-readiness/03-practitioner-guides/master-data-management.md) | AI use cases are diverging on customer, product, or entity records — you need a golden record |
| [Data Contracts](/enterprise-ai-transformation/tracks/03-data-readiness/03-practitioner-guides/data-contracts.md) | Upstream schema changes are breaking downstream models or pipelines without warning |
| [Synthetic Data Generation](/enterprise-ai-transformation/tracks/03-data-readiness/03-practitioner-guides/synthetic-data-generation.md) | You need training or test data but can't use production data due to privacy or volume constraints |
| [Data Labeling & Annotation Programs](/enterprise-ai-transformation/tracks/03-data-readiness/03-practitioner-guides/data-labeling-and-annotation-programs.md) | You are standing up a supervised learning or fine-tuning program and need quality labeled data at scale |
| [Data Mesh Governance in Practice](/enterprise-ai-transformation/tracks/03-data-readiness/03-practitioner-guides/data-mesh-governance-in-practice.md) | You are decentralizing data ownership but need to preserve governance consistency across domains |
| [Historical Data Debt — Pre-AI vs. Post-AI Remediation](/enterprise-ai-transformation/tracks/03-data-readiness/03-practitioner-guides/historical-data-debt-pre-ai-vs-post-ai-remediation.md) | You have legacy data that was never built for AI and need to decide what to fix, reframe, or abandon |
